# 記帳 Web APP 設計與功能規格書

這份文件用於記錄此 Mobile-first 個人記帳 Web APP 的所有設計規範、功能需求與後續的修改要求，以確保開發過程中的一致性。

## 1. 基礎架構與技術棧
- **框架**：Next.js (App Router) + React
- **樣式**：Tailwind CSS (v4)
- **圖示**：Lucide-React
- **字體**：Inter (強調英數字的易讀性，並搭配系統預設無襯線中文字體)

## 2. 開發協作流程 (Workflow)
未來所有的修改要求，將嚴格遵循以下三個步驟進行：
1. **檢視衝突**：檢查新的修改要求是否與本規格書（或原設計精神）有衝突，若有衝突會第一時間告知您。
2. **著手修改**：若無衝突，則直接進行程式碼與樣式的修改。
3. **更新規格書**：修改完成後，將該項要求與細節同步寫入本規格書中（置於對應章節與修訂紀錄）。

## 3. 設計規範 (iOS 26 Style)
### 2.1 材質與深度 (Material & Depth)
- **Ultra-thin Glassmorphism (深層級毛玻璃)**：
  - 卡片與面板需具備高強度的背景模糊 (blur-48px 以上)。
  - 邊框採用極細的半透明白線 (`border: 0.5px solid rgba(255,255,255,0.15)`)。
  - 需有發光內陰影 (Inner Glow) 的細節處理。
- **動態背景 (Dynamic Background)**：
  - 全域背景需有緩慢流動的漸層色彩與光暈 (Glowing Orbs)，並套用 Pulse 動畫效果。

### 2.2 排版與字體 (Typography)
- **核心數據與標題**：使用極大的字重 (font-[800])，並搭配緊湊的字距 (tracking-tight) 與等寬數字 (tabular-nums) 來呈現金額。
- **金額顯示 (台幣)**：所有金額數字均**取到整數**，不顯示小數點後方的位數 (例如 `$12,450` 而非 `$12,450.00`)。
- **純中文呈現**：介面上的所有資訊皆以**全中文**呈現，不顯示非必要之英文副標題 (如 Account Overview, Pending Transactions 等)。
- **時間與名詞本地化**：
  - `AM/PM` 轉換為「上午/下午」。
  - `Wallet` 轉換為「現金 / 錢包」。
  - 取消英文提示文字。

### 2.3 佈局與形狀 (Layout & Shapes)
- **卡片邊緣曲率**：所有主要卡片採用高曲率 (High Curvature) 配置，數值為 `rounded-[32px]` 或更高。
- **單行文字外框與標籤**：採用**膠囊狀** (`rounded-full`) 設計，確保左右對稱的完美半圓。
- **按鈕與圖示容器**：功能型圖示背景、登出等按鈕皆採用**完全圓形或膠囊狀** (`rounded-full`)。
- **全客製化彈窗**：不允許使用任何瀏覽器原生彈窗 (`alert`, `window.confirm`)，所有提示與防呆確認需以客製化 Modal 元件實作，以維持沉浸感與 iOS 26 視覺一致性。

## 4. 核心功能模組
### 3.1 底部導覽列 (Bottom Navigation Bar)
- **樣式**：懸浮設計 (Fixed Bottom)，整體外框呈**膠囊狀** (`rounded-full`)，具備毛玻璃材質與細邊框。
- **按鈕項目**：共 5 個選單 (帳戶、待審核、記帳/新增、報表、設定)。
- **新增按鈕 (Center Button)**：
  - 位於正中央，尺寸醒目。
  - **動態圖示切換**：不在首頁時顯示「房子 (Home)」圖示；而在首頁時則轉變為「加號 (+)」圖示，代表新增功能。
  - **佈局要求**：不往上凸出，必須與其他按鈕在同一個水平面上 (Inline)。
- **狀態切換**：點擊不同的 Tab 可無縫切換上方的視圖組件。

### 3.2 視圖 (Views) 要求
- **帳戶總覽 (AccountView)**：
  - **極簡卡片設計**：下方列表僅顯示「帳戶名稱」與「當前餘額」。
  - **新增與編輯帳戶**：
    - 切換為全螢幕視圖 (非傳統 Modal彈窗形式)。
    - **資產帳戶(Accounts)資料表定義：**
      - `id` (UUID): 唯一識別碼
      - `name` (String): 帳戶名稱
      - `initial_balance` (Decimal): 初始設定金額
      - `current_balance` (Decimal): 當前餘額 (系統自動計算)
      - `billing_cycle` (Integer): 帳單週期日 (1-31)
  - **明細與對帳模式**：點擊單一卡片進入詳細全螢幕頁面 (AccountDetailView)。包含完整明細列表、編輯與刪除功能。
  - **對帳邏輯**：點擊「對帳」按鈕後，依據該帳戶「帳單週期日」篩選當期明細。
  - **手勢操作 (Gestures)**：在明細項目上**右滑**：確認帳款 / **左滑**：延後入帳。
  - **帳務明細(Transactions)資料表定義：**
    - `id` (UUID): 唯一識別碼
    - `date` (DateTime): 交易日期
    - `type` (Enum): expense (支出), income (收入), transfer (內部轉帳)
    - `main_category` (String): 主分類
    - `sub_category` (String): 次分類
    - `account_id` (UUID): 關聯的帳戶 ID
    - `amount` (Decimal): 交易金額
    - `merchant` (String): 商家名稱
    - `invoice_number` (String): 發票號碼
    - `note` (String): 備註說明
    - `status` (Enum): confirmed (已確認), pending (待審核)
    - `billing_month` (String): 歸屬帳單月份 (YYYY-MM)，用於延後入帳邏輯
- **待審核 (PendingView)**：
  - **核心邏輯**：列出所有 `status: "pending"` 的帳務（來源包含 AI 辨識信心低、資料缺失或需手動核對者）。
  - **資料呈現**：列表極簡化，依日期分組顯示（如「今天」、「3月16日」），僅呈現「名稱 + 主次分類 + 金額」（不顯示具體時間點），並使用分類 Emoji 取代警示圖示。
  - **核對表單 (TransactionFormView)**：
    - **非捲動設計**：重新設計 UI 佈局，確保所有欄位在單一螢幕內完整呈現。
    - **欄位優先級**：將「日期、名稱、金額、主次分類」置於上方核心區域；「商家、帳單月、發票」等輔助資訊縮小呈現。
    - **類別選擇器進化**：主分類維持水平捲動膠囊，次分類改為緊湊的純文字標籤雲 (Chips)，移除圖示以節省空間。
    - **非捲動設計**：重新設計 UI 佈局，確保所有欄位在單一螢幕內完整呈現。
- **主頁 / 新增 (Home / Add)**：
  - **動態按鈕**：若用戶位於其他 Tab，中心按鈕顯示為「首頁」圖示；若已位於首頁，按鈕變更為「新增 (+)」圖示。
  - **首頁 (HomeView)**：
    - **月曆視圖**：上半部顯示極簡 iOS 風格月曆。支援月/週視圖切換（縮小時僅留一週），具備縮放動畫。
    - **每日明細**：下半部顯示選定日期的帳務列表。
    - **預算進度**：位於月曆下方，以極簡進度線呈現。
  - **新增 (AddView)**：
    - **介面**：單一大型自然語言輸入框，搭配相機與麥克風按鈕。
    - **功能**：支援最多 5 張影像解析。
- **預算管理 (BudgetView / EditBudgetView)**：
  - **入口**：位於首頁月曆與明細之間。
  - **功能**：顯示各主類別預算、已支出金額及百分比。
  - **編輯**：支援設定個別主類別預算及總預算加總計算。
- **分析報表 (ReportsView)**：
  - **篩選器**：支援時間範圍（月切換）與特定帳戶篩選。
  - **圖表**：採用高級感之圓環圖 (Donut) 或長條圖。
  - **下鑽邏輯 (Drill-down)**：點擊主分類後，圖表與列表動態切換至該類別下的「次分類」明細。
- **設定 (SettingsView)**：簡潔的系統工具介面。包含分類管理（主次類別編輯、按鈕、圖示與顏色設定）以及數據備份與還原工具。
